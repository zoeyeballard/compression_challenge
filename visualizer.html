02 — HOW AUDIO BECOMES NUMBERS

Sampling and quantization

A continuous voltage is measured (sampled) at a fixed rate, and each measurement is rounded to the nearest integer level (quantized). 16-bit means 2¹⁶ = 65,536 possible levels. Fewer bits = coarser steps = a staircer signal. Drag the bit-depth to see quantization error appear — and preview why throwing bits away is lossy.

03 — THE LOSSLESS RULE

The constraint that changes everything

The challenge requires lossless compression: decode(encode(x)) must equal x, bit for bit. That single rule removes the easiest 100× tricks — you cannot drop quiet bits, round, or model-and-approximate. Every last sample must come back exactly.

04 — ENTROPY: THE FLOOR

How small can it get?

Shannon's answer: a stream of symbols with probabilities pi needs at least H = −Σ pi log₂ pi bits per symbol, on average. That's the entropy — the hard floor no lossless coder can beat. A flat (uniform) distribution is incompressible; a peaked one is cheap. Reshape the distribution and watch the floor move.

05 — PREDICTION

Predict, then store only the surprise

Because neighbouring samples are similar, we can predict each one from the last and store just the residual Δ = xₙ − xₙ₋₁. If the guess is good, residuals cluster tightly around zero — a peaked distribution, which we just saw is cheap. 06 — THE QUANTIZATION CLUE

The data is hiding a secret

Delta coding got us partway, but the real breakthrough came from a histogram. Across an entire 98,689-sample file there are only 266 distinct sample values, and the deltas pile up on multiples of 64. The signal was quantized upstream — which means the "alphabet" is tiny, and a model that learns those exact values can do far better than any generic byte compressor.

07 — CONTEXT MODELING

The level predicts the next move

Even better: which delta comes next depends on where the signal currently is. Conditioning each delta on the quantized previous level (prev >> 6) drops the entropy from ~5.2 down to ~4.2 bits/sample — an "order-1" model. 08 — THE RANGE CODER

Turning probabilities into bits

Knowing the floor isn't enough — we need a coder that actually hits it. A range (arithmetic) coder represents the whole message as a single fraction inside [0,1). Each symbol narrows the interval to its probability slice; likely symbols barely shrink it (cheap), rare ones shrink it a lot (expensive). The final interval's width is exactly 2^(−bits used). 09 — ESCAPES (PPM-C)

What if a symbol is brand new?

A fresh context has never seen most values. The codec uses a two-level fallback: try the context model first; if the symbol isn't there, emit a cheap escape and drop to a global model; if it's new there too, escape again to a raw 17-bit code. 11 — TRAPS & CONSTRAINTS

Where it gets hard

Getting to 3.3× meant walking into — and out of — several traps. Each one below is a real decision point in the build, with a demo of the failure mode.

Trap · dead end (reached only 1.6×)

Splitting a value into "size + raw bits" assumes the bits are random

A natural scheme codes a delta's magnitude class, then its lower "mantissa" bits as-is. That's near-optimal for smooth noise — but this data's values cluster on specific levels, so the mantissa bits are not uniform. The scheme throws away exactly the structure that matters. Constraint · the adaptation cliff

More context looks better on paper — until the data runs out

The ideal entropy always drops as you add context bits. But an adaptive coder must learn each context from real samples. Split 98k samples across too many contexts and each is starved, so the realized ratio peaks and then falls. This is exactly why adding a sign bit or a second sample to the context made things worse in testing.

Constraint · integer coder limits

The frequency totals must stay under 2¹⁶

A 32-bit range coder loses precision if the model's total count grows too large. So counts are halved whenever they hit the cap — which conveniently also lets the model "forget" and track drift. Watch the total climb and rescale.

Constraint · the 200× wall

The headline target is physically out of reach — losslessly

200× would mean 0.08 bits per sample. But the data's own entropy is ~4.2 bits/sample: that's the floor, and no lossless method can go below it. 200× lives in lossy territory. The honest lossless target is the entropy ceiling, ~3.3–3.6×.
